#[3] Etapa del proyecto: [Etapa 3] Fecha:19-05-2026
 ------------------------------------------------------------
PROMPT ENVIADO A COPILOT: [Actúa como un desarrollador front-end experto en JavaScript vanilla.

Estoy trabajando en la Etapa 3 del proyecto VetCare. Ya tengo creada la página registro.html con su formulario y estilos CSS. Ahora necesito crear el archivo validaciones.js y conectarlo correctamente con registro.html.

Debes usar solo JavaScript nativo. No uses librerías externas.

Objetivo:
Validar el formulario de registro del dueño y la mascota antes de enviarlo, mostrar errores visuales y, si todo está correcto, mostrar una pantalla de confirmación.

Requisitos principales:

1. Interceptar el envío del formulario usando submit y preventDefault().

2. Validar los datos del dueño:
   - Nombre completo: solo letras y espacios, mínimo 5 y máximo 80 caracteres.
   - Fecha de nacimiento: el dueño debe ser mayor de 18 años.
   - DNI: solo números, entre 7 y 8 dígitos.
   - Correo electrónico: formato válido con expresión regular.
   - Confirmar correo: debe coincidir con el correo.
   - Contraseña: mínimo 8 caracteres, una mayúscula, un número y un carácter especial.
   - Confirmar contraseña: debe coincidir con la contraseña.
   - Teléfono: permite números, espacios, guiones y signo +, con mínimo 8 dígitos.
   - Dirección y localidad: obligatorias, mínimo 3 caracteres.
   - Provincia: debe seleccionarse una opción válida.

3. Validar los datos de la mascota:
   - Nombre de la mascota: obligatorio, solo letras y espacios, mínimo 2 caracteres.
   - Especie: debe seleccionarse una opción válida.
   - Si especie es “Otro”, mostrar un campo adicional y validarlo como obligatorio.
   - Raza: obligatoria, mínimo 2 caracteres.
   - Sexo: debe seleccionarse una opción.
   - Fecha de nacimiento: no puede ser futura y la mascota no puede tener más de 30 años.
   - Si el checkbox de microchip está marcado, mostrar el campo número de microchip y validar entre 9 y 15 dígitos.
   - Alergias: máximo 250 caracteres.

4. Validar preferencias de atención:
   - Al menos un servicio de interés debe estar seleccionado.
   - Turno preferido obligatorio.
   - Frecuencia de visitas obligatoria.

5. Validar confirmación:
   - Observaciones: máximo 300 caracteres.
   - Términos y Condiciones obligatorio.
   - Política de Privacidad obligatoria.

6. Retroalimentación visual:
   - Si un campo tiene error, agregar la clase campo-error.
   - Si un campo está correcto, agregar la clase campo-ok.
   - Mostrar mensajes de error debajo de cada campo usando JavaScript.
   - Eliminar el mensaje de error cuando el usuario corrija el campo.
   - Usar eventos input, change y blur para validar en tiempo real.

7. Funcionalidades dinámicas:
   - Mostrar u ocultar el campo “Otro” según la especie seleccionada.
   - Mostrar u ocultar el campo número de microchip según el checkbox.
   - Agregar contador de caracteres en tiempo real para alergias y observaciones.
   - El contador debe cambiar visualmente al superar el 80% del límite.

8. Al enviar el formulario:
   - Mostrar un resumen al inicio con la cantidad de errores encontrados.
   - Hacer scroll automático al primer campo con error usando scrollIntoView().
   - Si no hay errores, ocultar el formulario y mostrar una pantalla de confirmación.

9. Pantalla de confirmación:
   - Debe mostrar el nombre del dueño.
   - Debe mostrar el nombre de la mascota.
   - Debe generar un número de registro aleatorio con Math.random().
   - Debe mostrar los servicios seleccionados.
   - Debe incluir un botón “Volver al inicio”.
   - Debe incluir un botón “Registrar otra mascota” que reinicie o recargue el formulario.

Entrega:
Genera el código completo de validaciones.js y dime exactamente dónde debo enlazarlo dentro de registro.html usando la etiqueta script.

Mantén el código claro, ordenado, comentado solo cuando sea necesario y fácil de entender para un estudiante.] ------------------------------------------------------------
